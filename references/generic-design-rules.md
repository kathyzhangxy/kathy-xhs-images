## 规则 1：同行元素对齐规范

**适用场景**：同一行内多个文字元素的对齐（标题+标签、导航菜单、按钮组等）

### 对齐方式选择

| 对齐方式 | 适用场景 | 说明 |
|---------|---------|------|
| **底部基线对齐** | 字号不同的文字同行显示 | **默认选择**。大字和小字底部在同一水平线，小字顶部自然更低 |
| **垂直居中对齐** | 图标+文字、按钮内文字 | 元素整体垂直居中 |
| **顶部对齐** | 特殊设计需求 | 较少使用，需明确说明 |

### Prompt 描述规范

```
✅ 正确：右上角标签"文字"（样式：小号16px，右对齐，与主标题底部基线对齐）
❌ 错误：右上角标签"文字"与主标题在同一水平线（模糊，ImageGen 可能理解为垂直居中）
```

### 视觉判断标准

- 底部基线对齐时，由于字号差异，小字的顶部会自然低于大字顶部
- **这是正常且美观的效果**，不是对齐问题
- Evaluator 检查时应关注底部是否对齐，而非顶部

## 规则 2：长文本一行展示方案库

**适用场景**：标题/标签过长，一行空间有限

### 方案对比

| 方案 | 策略 | 适用条件 | 视觉影响 |
|------|------|---------|---------|
| **方案 A：缩小主标题** | 主标题字号递减，标签保持 | 主标题是重点，标签是辅助 | 主标题变小，但层次关系保持 |
| **方案 B：缩小标签** | 主标题保持，标签字号递减 | 主标题不能缩小 | 标签更不显眼 |
| **方案 C：换行** | 长文本拆分为两行 | 超过画布宽度 80% | 布局高度增加 |
| **方案 D：省略** | 长文本截断+省略号 | 空间极度受限 | 信息不完整 |
| **方案 E：调整布局** | 改为上下结构 | 同行无法容纳 | 布局改变 |

### 字号递减参考表

| 主标题字符数 | 主标题字号 | 标签字号 | 备注 |
|-------------|-----------|---------|------|
| ≤8 英文 / ≤6 中文 | 44px | 16px | 正常布局 |
| 9-12 英文 / 7-8 中文 | 40px | 15px | 适度缩小 |
| 13-18 英文 / 9-12 中文 | 36px | 14px | 明显缩小 |
| >18 英文 / >12 中文 | 32px | 13px | 显著缩小 |

### 长标题处理决策树

```
主标题过长？
├─ 是 → 字符数是多少？
│   ├─ ≤12 英文 / ≤8 中文 → 方案：适度缩小（40px）
│   ├─ 13-18 英文 / 9-12 中文 → 方案：明显缩小（36px）
│   └─ >18 英文 / >12 中文 → 方案：显著缩小（32px）或考虑换行
│
└─ 否 → 使用标准字号（44px）
```

### Prompt 描述规范

```
主标题："Agent Browser"（样式：大号粗体，深灰色 #333333，字号40px，左对齐。
**标题较长(13字符)，从标准44px缩小至40px，为右上角标签腾出空间**）
右上角："网页自动化利器"（样式：小号文字，浅灰色 #999999，字号15px，右对齐，
与主标题底部基线对齐）
```

## 规则 3：Prompt 结构化通用模板

**核心原则**：样式元数据与内容文字必须明确分离，ImageGen 才能正确理解

### 3.1 标准结构

```
"内容文字"（样式：属性1，属性2，属性3）
```

### 3.2 结构拆解

| 部分 | 格式 | 示例 | 说明 |
|------|------|------|------|
| 内容文字 | `"引号包裹"` | `"GitHub"` | ImageGen 渲染到图片上的文字 |
| 样式描述 | `（样式：...）` | `（样式：大号粗体，深灰色 #333333，字号44px）` | 样式参数，不渲染到图片 |

### 3.3 样式属性分类

| 属性类别 | 包含内容 | 示例 |
|---------|---------|------|
| **尺寸** | 字号、直径、间距、宽度 | `字号44px`、`直径约45px`、`间距8px` |
| **颜色** | 色值、颜色名称 | `深灰色 #333333`、`橙色 #ed8936` |
| **字重** | 常规、粗体、超粗体 | `粗体`、`超粗体` |
| **字体** | 等宽、常规、手写 | `等宽字体 monospace` |
| **对齐** | 左对齐、右对齐、居中 | `左对齐`、`右对齐` |
| **位置关系** | 对齐基准、相对位置 | `与主标题底部基线对齐` |

### 3.4 正确 vs 错误写法对比

| 场景 | ✅ 正确写法 | ❌ 错误写法 | 错误原因 |
|------|-----------|-----------|---------|
| 文字+样式 | `"GitHub"（样式：大号粗体，深灰色 #333333，字号44px，左对齐）` | `"GitHub"大号粗体44px左对齐` | 样式未被括号包裹，ImageGen 可能渲染部分样式 |
| 尺寸参数 | `直径约45px`（在样式括号内） | `"直径45px"` | px 值被引号包裹，会被渲染为文字 |
| 颜色值 | `深灰色 #333333` | `"#333333"` | 色值不应独立成内容 |
| 对齐描述 | `与主标题底部基线对齐` | `"底部基线对齐"` | 对齐描述是样式元数据，不是内容 |

### 3.5 区块结构化模板

```
【区域名称】（约XX%高度）
元素1："内容1"（样式：属性列表）
元素2："内容2"（样式：属性列表）
底部/分隔："说明"（样式：属性列表）
```

### 3.6 完整示例

```
【顶部标题区】（约15%高度）
主标题："Self-Improving Agent"（样式：中号粗体，深灰色 #333333，字号36px，左对齐。**标题超长(21字符)，从标准44px缩小至36px**）
右上角："持续进化引擎"（样式：小号文字，浅灰色 #999999，字号14px，右对齐，与主标题底部基线对齐）
底部：细线分隔（样式：暖灰色 #d0c8bc，1px细线，横跨全宽）
```

## 规则 4：ImageGen 理解能力边界

### ImageGen 能正确理解的

| 类型 | 示例 | 说明 |
|------|------|------|
| 结构化格式 | `"内容"（样式：描述）` | 内容与样式明确分离 |
| 明确的位置关系 | "左对齐"、"右对齐"、"底部基线对齐" | 使用标准术语 |
| 括号内的尺寸参数 | `字号44px`、`直径约45px` | 在样式括号内 |
| 颜色描述 | `深灰色 #333333` | 颜色名称+色值 |

### ImageGen 容易误解的

| 类型 | 错误示例 | 正确替代 |
|------|---------|---------|
| 模糊的对齐描述 | "在同一水平线" | "底部基线对齐" |
| 样式和内容混在一起 | `"44px粗体文字"` | `"文字"（样式：字号44px，粗体）` |
| 多个元素未明确关系 | "A 和 B" | "A 在左，B 在右，间距8px" |
| 负向描述 | "不要渲染X" | 直接不写 X |

## 规则 5：Evaluator 检查清单（通用版）

### A 类：Prompt 结构化

- [ ] 所有"要在图片上显示的文字"是否用引号包裹？
- [ ] 所有样式描述是否放在 `（样式：...）` 括号内？
- [ ] px 值、色值等参数是否只在样式括号内出现？
- [ ] 样式属性是否按分类组织（尺寸/颜色/字重/字体/对齐）？

### B 类：对齐一致性

- [ ] 同行元素是否使用正确的对齐方式（默认底部基线对齐）？
- [ ] 长标题是否应用了字号递减规则？
- [ ] Prompt 中对齐描述是否使用了精确术语（"底部基线对齐"而非"同一水平线"）？
- [ ] 右上角标签底部是否与主标题底部在同一水平线？

### C 类：布局一致性

- [ ] 页面结构是否与选定布局模板一致？
- [ ] 各区域高度比例是否合理？
- [ ] 元素间距是否有明确定义？

### D 类：完整性检查

- [ ] 所有文案内容是否完整无缺失？
- [ ] 列表项数量是否正确？
- [ ] 统计数据是否完整？

### E 类：命名规范

- [ ] 文件命名是否符合 `P{N}-{主题}-V{版本}.png` 规范？
- [ ] 存放位置是否正确？

## 规则汇总速查表

| 规则 | 一句话总结 | 应用场景 |
|------|-----------|---------|
| **同行对齐** | 默认使用底部基线对齐 | 标题+标签、导航、按钮组 |
| **长文本处理** | 5种方案可选，优先缩小字号 | 标题过长、标签过长 |
| **Prompt结构化** | `"内容"（样式：元数据）` | 所有 prompt 编写 |
| **样式分类** | 尺寸/颜色/字重/字体/对齐 | 样式描述组织 |
| **ImageGen边界** | 模糊描述会误解，精确术语才可靠 | prompt 措辞选择 |
| **Evaluator检查** | 5类清单逐项检查 | 质量审查 |
