## 示例 1：浏览器预览 + 功能清单（P1-V8）

**布局类型：** A
**页面类型：** Inner

```
实战案例：用豆包做一个歌词日历网页！蓝色超大号粗体（48px）标题，字号明显大于正文（18px），位于页面顶部。

页面内容分为2个区域：
1. 浏览器预览窗口：模拟浏览器窗口，地址栏显示 file:///calendar.html，窗口内展示歌词日历效果（每日经典歌词日历标题、2026年4月3日日期、歌词卡片显示歌名和歌手），用蓝色手绘线条框组织，配手绘浏览器图标
2. 功能清单：用绿色手绘对勾图标列出5个功能点（显示当前日期：2026-04-03、展示当日歌词：歌名+歌手+歌词、左右翻页：切换前一天/后一天、风格切换：摇滚/港台/周杰伦/民谣、切换风格后，歌词数据全部更新）

纯白背景。手绘风格，蓝色墨水笔线条。标题蓝色，内容用蓝色手绘线条框组织，不要实色填充背景。用手绘小图标代替emoji。颜色克制，主要使用蓝色，最多1个绿色点缀。不要画页码、系列名、网格线、徽章。
```

**成功要素：**
- 标题醒目，"实战案例："前缀 + 超大号
- 浏览器预览 + 功能清单左右并列
- 绿色对勾点缀，色彩有层次

## 示例 2：手机双屏对比（P3-V5）

**布局类型：** B
**页面类型：** Inner

```
第1步：给豆包发指令！蓝色超大号粗体（48px）标题，字号明显大于正文（18px），位于页面顶部。副标题"把 AI 生成的 Prompt 稍做优化，直接发给豆包"用较小字号，蓝色。

页面内容分为左右两个手机屏幕：

左侧手机屏幕：模拟豆包 App 界面，顶部状态栏显示时间，标题栏显示"豆包"，中间蓝色渐变对话气泡展示发送给豆包的结构化 Prompt 文本（角色设定、功能需求、样式要求、数据要求、输出格式），白色等宽字体，底部豆包操作栏

右侧手机屏幕：模拟豆包 App 界面，顶部状态栏显示时间，标题栏显示"豆包"，中间白色圆角窗口展示代码输出界面，顶部"预览|代码"标签切换，窗口内展示HTML代码片段，底部豆包操作栏

中间用粗手绘曲线箭头从左手机指向右手机，带手绘曲线感。

底部区域：手绘风格蓝色虚线框，框内手绘灯泡图标 + "发送后等待10-30秒，豆包会生成完整HTML代码"，其中"10-30秒"用橙色手绘圆圈圈出。

纯白背景。手绘风格，蓝色墨水笔线条。
颜色层次：主色蓝色 + 点缀橙色 + 次要色紫色。
点缀元素：右上角手绘小星星（橙色）、左侧窗口旁手绘对话气泡小图标（紫色）、右侧窗口旁手绘代码标签小图标（橙色）、"10-30秒"橙色手绘圆圈。
不要实色填充大区域，只在局部点缀。点缀色4-5个元素。
用手绘小图标代替emoji。不要画页码、系列名、网格线、徽章。
页面内容从上到下均匀分布，不要集中在上半部分。
```

**成功要素：**
- 手机屏幕完整模拟（状态栏+标题栏+内容区+底部操作栏）
- 中间手绘曲线箭头连接
- 底部手绘虚线框替代实心色块
- 4-5个点缀元素分散在多处

## 使用指南

1. 根据页面类型选择对应的布局类型（A/B/C）
2. 复制对应示例作为基础模板
3. 替换具体内容（标题、功能点、App名称等）
4. 根据实际需要调整点缀元素数量和类型
5. 参考 `references/layouts/layout-patterns.md` 获取更多布局模板
