## 布局 A：浏览器预览 + 功能清单

```
┌─────────────────────────────────┐
│  [标题]                          │ ← 超大标题
├─────────────────────────────────┤
│  ┌─────────────────┐  ☑ 功能1    │
│  │  浏览器预览窗口  │  ☑ 功能2    │
│  │                 │  ☑ 功能3    │
│  │  内容展示区      │  ☑ 功能4    │
│  │                 │  ☑ 功能5    │
│  └─────────────────┘             │
├─────────────────────────────────┤
│  ✅ [底部提示]                    │ ← 底部提示
└─────────────────────────────────┘
```

**适用场景：** P1-任务介绍、P3-需求说明、P10-最终效果展示

**Prompt 模板（风格无关结构）：**
```
[标题]！[风格.标题样式]，位于页面顶部。

页面内容分为2个区域：
1. 浏览器预览窗口：模拟浏览器窗口，地址栏显示 [URL]，窗口内展示[具体内容]，用[风格.线条风格]框组织，配手绘浏览器图标
2. 功能清单：用[风格.功能图标]列出[数量]个功能点（[功能1]、[功能2]...）

[风格.约束模板]
页面内容从上到下均匀分布，不要集中在上半部分。
```

**铅笔手绘风示例：**
```
实战案例：用豆包做一个歌词日历网页！蓝色超大号粗体（48px）标题，字号明显大于正文（18px），位于页面顶部。

页面内容分为2个区域：
1. 浏览器预览窗口：模拟浏览器窗口，地址栏显示 localhost:3000，窗口内展示歌词日历网页效果，用蓝色手绘线条框组织，配手绘浏览器图标
2. 功能清单：用绿色手绘对勾图标列出5个功能点（自动生成歌词、日历样式、多歌曲切换、杨千嬅紫配色、手机自适应）

纯白背景。手绘风格，蓝色墨水笔线条。标题蓝色，内容用蓝色手绘线条框组织，不要实色填充背景。用手绘小图标代替emoji。颜色克制，主要使用蓝色，最多1个绿色点缀。不要画页码、系列名、网格线、徽章。
```

## 布局 B：手机双屏对比

```
┌─────────────────────────────────┐
│  [标题]                          │ ← 超大标题
│  [副标题]                        │ ← 较小字号
├─────────────────────────────────┤
│  ┌──────────┐    ┌──────────┐  │
│  │  左手机屏 │ → │  右手机屏 │  │ ← 手绘箭头
│  │  (输入)   │    │  (输出)   │  │
│  └──────────┘    └──────────┘  │
├─────────────────────────────────┤
│  [底部提示框]                    │ ← 底部提示
└─────────────────────────────────┘
```

**适用场景：** 展示"发送指令→AI生成"流程、"修改前→修改后"对比

**Prompt 模板（风格无关结构）：**
```
[标题]！[风格.标题样式]，位于页面顶部。副标题"[副标题文案]"用[风格.副标题样式]。

页面内容分为左右两个手机屏幕：

左侧手机屏幕：模拟 [App名称] App 界面，顶部状态栏显示时间，标题栏显示"[App名称]"，中间[风格.内容容器]展示[具体内容]

右侧手机屏幕：模拟 [App名称] App 界面，顶部状态栏显示时间，标题栏显示"[App名称]"，中间[风格.内容容器]展示[具体内容]

中间用[风格.连接箭头]从左手机指向右手机。

底部区域：[风格.提示框样式]，框内[风格.图标] + "[提示文字]"，其中"[关键词]"用[风格.高亮样式]标出。

[风格.约束模板]
[风格.点缀模板]
页面内容从上到下均匀分布，不要集中在上半部分。
```

**关键成功要素：**
- ✅ 用"手机屏幕"描述，不用"对话气泡"或"圆角矩形"
- ✅ 每个屏幕包含：状态栏 + 标题栏 + 内容区 + 底部操作栏
- ✅ 中间用"粗手绘曲线箭头"连接，强调"手绘曲线感"
- ✅ 底部提示用风格化提示框，不用固定图标类型
- ✅ 点缀分散在多处（右上角、左侧、右侧、关键词）
- ✅ 副标题文案要口语化

## 布局 C：操作步骤时间线（P4-V2 已验证）

```
┌─────────────────────────────────┐
│  [标题]                          │ ← 超大标题
│  [副标题]                        │ ← 较小字号
├─────────────────────────────────┤
│  ┌───────────────────────────┐  │
│  │ ① [图标] 步骤1说明          │ ← [风格.普通步骤框]
│  └───────────────────────────┘  │
│         ↕ [风格.连接元素]        │
│  ┌───────────────────────────┐  │
│  │ ② [图标] 步骤2说明          │ ← [风格.普通步骤框]
│  └───────────────────────────┘  │
│         ↕ [风格.连接元素]        │
│  ┌───────────────────────────┐  │
│  │ ③ [图标] 步骤3说明          │ ← [风格.强调步骤框]
│  └───────────────────────────┘  │
│         ↕ [风格.连接元素]        │
│  ┌───────────────────────────┘  │
│  │ ④ [图标] 步骤4说明          │ ← [风格.普通步骤框]
│  └───────────────────────────┘  │
└─────────────────────────────────┘
```

**适用场景：** 多步操作教程、跟着做系列

**关键成功要素：**
- ✅ **每个步骤必须用边框容器包裹**，否则内容会显得散乱
- ✅ 内容描述用"框内左侧/框内右侧/框内上方/框内下方"结构
- ✅ 普通步骤用风格主色框，需要强调的关键步骤用风格强调色框
- ✅ 步骤间用风格连接元素连接（上一个框底部指向下一个框顶部）
- ✅ 每个步骤左侧放编号圆圈，右侧放图标+文字

**Prompt 模板（风格无关结构）：**
```

- 中间：文字"[步骤说明文字]"

框内右侧：
- 手绘[图标]（[图标描述]）

第N步（[风格.强调步骤框]包裹，重点强调）

- 上方：手绘[图标]（[图标描述]）

[风格.点缀模板]
页面内容从上到下均匀分布，不要集中在上半部分。如果内容较少，在底部添加装饰性总结框或提示框，撑满整图。

│  └────┘  └────┘  └────┘       │
│  ← [风格.标注] →                │
├─────────────────────────────────┤
│  ┌───────────────────────────┐  │

│  └───────────────────────────┘  │
│  ┌───────────────────────────┘  │
│  │ 💻 方法2（[风格.方案卡片]）   │ ← 卡片2（内嵌代码）

```

**适用场景：** 问题调试页、错误修复指导、优化建议

**关键成功要素：**
- ✅ **顶部问题场景**：用日历格/对比图等方式直观展示问题，不要加区域标题文字
- ✅ **风格化标注**：用风格强调色 + 箭头 + 文字标注问题位置
- ✅ **方法卡片竖向排列**：每个方法卡片内可嵌入代码块
- ✅ **去掉底部单独模块**：如果方法内已包含完整示例，不需要额外底部模块

- ✅ **冠军页特殊处理**：右上角👑+冠军，底部浅橙色卡片+🏆+整合数据
- ✅ **所有文字内容用引号包裹**，样式描述放在括号内，避免 ImageGen 将样式值渲染为文字

- ✅ **手机屏幕下方截断**：只展示到歌词/核心内容区域即可，不需要完整手机底部
- ✅ **手机屏幕撑满左右**：2x2 网格中每个手机屏幕宽度要撑满画布左右两侧，不要左右留白

- ✅ **多状态展示**：如需展示不同状态（如不同 tab 选中），确保每个手机屏幕的状态标注清晰
