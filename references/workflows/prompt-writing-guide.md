## 风格注入（编写 Prompt 前的必要步骤）

**在编写 Prompt 之前，必须确定当前使用的风格和画布，并注入其约束。**

### Step 1：确定风格

当前默认风格为 `pencil-handdrawn`（铅笔手绘风）。风格定义详见 `references/styles/pencil-handdrawn.md`。

从风格定义中提取以下关键参数：
- **主色、点缀色、强调色、功能色**
- **线条风格描述**
- **约束模板**（如"纯白背景。手绘风格，蓝色墨水笔线条..."）
- **点缀模板**（如"点缀元素（分布在页面各处...）"）

### Step 2：确定画布

当前默认画布为 `grid-canvas`（网格画布）。画布定义详见 `references/canvases/grid-canvas.md`。

从画布定义中提取：
- **宽高比**（3:4）
- **安全区域**（顶部60px、左右50px、底部80px）

### Step 3：注入到 Prompt

在布局模板的基础上，将所有 `[风格.XXX]` 占位符替换为实际风格参数。

**替换示例（铅笔手绘风）：**
- `[风格.标题样式]` → "蓝色超大号粗体（48px）标题，字号明显大于正文（18px）"
- `[风格.边框容器]` → "手绘蓝色圆角矩形框"
- `[风格.主色]` → "蓝色"
- `[风格.点缀色1]` → "橙色"
- `[风格.约束模板]` → "纯白背景。手绘风格，蓝色墨水笔线条。标题蓝色，内容用蓝色手绘线条框组织，不要实色填充背景。用手绘小图标代替emoji。颜色克制，主要使用蓝色，最多1个绿色点缀。不要画页码、系列名、网格线、徽章。"

## 三段式结构

```
[标题]：[风格.标题样式]，[位置描述]。

[区域描述]：
1. [区域1名称]：[具体内容描述]，用[风格.边框容器]组织，配手绘[图标]
2. [区域2名称]：[具体内容描述]，用[风格.边框容器]组织，配手绘[图标]
...

[风格.约束模板]
[风格.点缀模板]
页面内容从上到下均匀分布，不要集中在上半部分。
```

## 风格约束模板

风格约束模板定义在 `references/styles/[风格名].md` 中。当前默认风格 `pencil-handdrawn` 的约束模板为：

```
纯白背景。铅笔手绘线条风（notion风格），蓝色铅笔线条。
标题蓝色，内容用蓝色手绘线条框组织，不要大面积实色填充背景。
用手绘小图标代替emoji。
颜色克制，主要使用蓝色，点缀橙色和紫色。
不要画页码、系列名、网格线、徽章。
页面内容从上到下均匀分布，不要集中在上半部分。
```

**未来新增风格时**，需要在风格定义文件中提供对应的约束模板，编写 Prompt 时直接引用即可。

## 标题规范

| 规则 | 说明 | 示例 |
|------|------|------|
| 必须加"实战案例："前缀 | 所有实战类页面 | "实战案例：用豆包做一个歌词日历网页！" |
| 必须指定超大号 | 标题字号明显大于正文 | "蓝色超大号粗体（48px）" |
| 必须对比正文字号 | 明确说明与正文差异 | "字号明显大于正文（18px）" |
| 标题位置 | 位于页面顶部 | "位于页面顶部" |

## 内容撑满技巧

当内容较少时，prompt 中必须包含以下描述之一：
- "页面内容从上到下均匀分布，不要集中在上半部分"
- "底部添加补充说明区域，用蓝色手绘线条框组织"
- "主视觉元素适当放大，占据页面主要垂直空间"
- "如果内容较少，在底部添加装饰性总结框或提示框，撑满整图"

## 手绘元素分类

**重要原则：区分有意义的手绘元素 vs 无意义的点缀元素**

| 类型 | 定义 | 摆放规则 | 示例 |
|------|------|---------|------|
| **有意义的手绘元素** | 与内容强相关，承载信息传达功能 | 必须根据内容编排放置在指定位置 | 编号圆圈、步骤图标、警告三角、代码块框、连接虚线、箭头、关键词圆圈 |
| **无意义的点缀元素** | 纯装饰性，增加画面层次感和趣味性 | 适度增加，分散在页面各处，不局限在固定位置 | 小星星、小彩带、小圆圈、小点、小线条 |

### 有意义的手绘元素（必须精准定位）

这类元素是**内容的一部分**，prompt 中必须明确描述其位置和作用：

- **编号圆圈**：步骤类页面，每个步骤左侧，"左侧手绘圆形编号'1'（蓝色粗边框）"
- **步骤图标**：与步骤内容强相关，"手绘复制图标（两个重叠的矩形）"
- **警告/提示图标**：关键信息旁边，"旁边配手绘警告三角图标"
- **连接元素**：模块之间，"四步之间用蓝色手绘虚线连接"
- **关键词高亮**：重要文字周围，"'10-30秒'用橙色手绘圆圈圈出"
- **框线/容器**：内容组织，"用浅蓝色手绘线条框展示代码块"

### 无意义的点缀元素（适度增加，自由分布）

这类元素是**纯装饰**，prompt 中描述数量和分布范围即可，不需要精确位置：

**点缀分布原则：**
- **数量**：8-15个点缀元素
- **位置**：分散在画面各处，不要集中在一个位置或只放在右上角
- **颜色**：主色蓝色 + 点缀橙色 + 次要色紫色
- **形式**：多种类型组合，不要只用一种

**可用的点缀类型：**
1. **手绘小星星** — 2-3个，橙色，"手绘橙色小星星（2-3个，分散在页面空白处）"
2. **手绘小彩带** — 1-2条，橙色或紫色，"手绘彩色小彩带（1-2条，斜向飘落）"
3. **手绘小圆圈** — 3-4个，蓝色或橙色，大小不一，"手绘小圆圈（3-4个，大小不一，散布在步骤之间）"
4. **手绘小点** — 5-6个，蓝色，"手绘小点（5-6个，随机分布在空白区域）"
5. **手绘小线条** — 2-3条，蓝色或橙色，"手绘短线条（2-3条，随机角度）"

**Prompt 写法示例：**
```
点缀元素（分布在页面各处，不要集中在一个位置）：
- 手绘橙色小星星（2-3个，分散在页面空白处）
- 手绘彩色小彩带（1-2条，橙色或紫色，斜向飘落）
- 手绘小圆圈（3-4个，蓝色或橙色，大小不一，散布在步骤之间）
- 手绘小点（5-6个，蓝色，随机分布在空白区域）
```

## 常见错误

| 错误 | 表现 | 正确做法 |
|------|------|---------|
| 结构标签当内容 | ImageGen 把"区域1"等标签画出来 | 用自然语言描述位置："左上方"、"底部"等 |
| 底部留白过多 | 内容集中在上半部分 | 明确写"均匀分布"，添加底部装饰区域 |
| 标题不够醒目 | 标题颜色淡或模糊 | 写"醒目的深蓝色粗体大号标题" |
| 不区分有意义/无意义元素 | 装饰元素出现在错误位置，或内容图标散乱分布 | 有意义元素必须指定位置，无意义点缀只描述数量和分布范围 |
| 点缀只放右上角 | 画面一侧空洞，一侧拥挤 | 点缀分散在页面各处，明确写"不要集中在一个位置" |
| 包含不必要模块 | 页面信息过载，主次不分 | 根据内容重要性删减模块（如操作步骤页不需要常见问题） |
| 步骤内容无边框容器 | 内容散落在页面上，显得散乱无结构 | 每个步骤用手绘圆角矩形框包裹，用"框内左侧/右侧/上方/下方"描述内容布局 |
| 预览页标注问题 | 纯预览页加红色箭头和问题清单，显得冗余 | 预览页只展示美观效果，问题标注放到下一页专门讲问题 |
| 方案页底部重复示例 | 方案卡片内已有完整示例，底部再加代码块模块导致信息重复 | 如果方案内已包含完整内容，去掉底部额外示例模块 |
| 多方案卡片横向排列拥挤 | 卡片内容多时横向排列显示不全 | 改为竖向排列，确保每个方案内容完整展示 |
| 区域标题标签被画出 | 使用"【区域名】"格式会被 ImageGen 当作可见文字渲染 | 去掉区域标题，直接用自然语言描述位置 |
| 问题描述不准确 | 如"重复"未说明是歌曲重复还是歌词重复 | 明确标注问题类型，如"歌曲重复！" |
| 长条框被并排成多列 | 底部功能清单的长条框被渲染为横向多列排列 | 明确写"从上到下依次竖向排列，每个横向撑满页面宽度，不要并排成多列" |
| Tab 选中态双重边框 | Tab 选中态已有实色填充，外面又画了边框导致冗余 | 明确写"实色填充选中（外面不要边框）" |
| 手机预览留白过多 | 多宫格布局中手机屏幕左右两侧大量留白 | 明确写"手机屏幕撑满画布左右两侧，左右基本无留白" |
