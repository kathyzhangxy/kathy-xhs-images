# kathy-xhs-images Skill

> 小红书图片生成（ImageGen 直接调用流程）
> 版本：v1.0
> 创建日期：2026-04-04

## 概述

封装 Kathy 在小红书图片生成中积累的 ImageGen 直接调用流程。适用于需要完全控制 prompt 内容、精细调整每个视觉元素、或 baoyu skill 生成效果不满意的场景。

**触发条件：** 用户选择"方式 B：直接调用 ImageGen"，或在 Step D 中明确要求手动编写 prompt。

## 核心工作流

### Step 0：生图前确认（硬性规则）

每页调用 ImageGen 之前，必须执行以下两步：

#### 0.1 判断页面类型

| 类型 | 判断依据 | 参考案例目录 | 生成策略 |
|------|---------|-------------|---------|
| **Cover** | C2 中 P0 大字报封面 | `knowledge/cases/good/cover/` | 纯色背景+超大标题+关键词高亮，极简 |
| **Inner** | C2 中 P1-Pn 内容页 | `knowledge/cases/good/inner/` | 模块化布局+图标辅助+信息密度高 |

#### 0.2 汇报生图思路

汇报格式：
```
P{N} - {页面主题} 生图思路
- 页面类型：Cover / Inner
- 参考案例：good/cover 或 good/inner 中的具体文件

【布局框架】
{区域划分 + 各区域占比}

【内容模块】
{各模块文字内容 + 展现形式 + 图标}

【视觉元素】
{配色 + 装饰元素}

请确认是否开始生成？
```

**等待确认：** 必须等待用户明确回复"开始"或"确认"后才可调用 ImageGen。

### Step 1：复杂页面先确认结构（硬性规则）

当页面包含 2 个及以上独立视觉区域时，必须先用文字描述页面的布局结构和每个区域的表达方式，与用户确认后再调用 ImageGen。

简单页面（单一主题、单一视觉元素）可以直接生成。

### Step 2：编写 Prompt

#### 三段式结构

```
[标题]：[颜色][字号][粗细]标题，[位置描述]。

[区域描述]：
1. [区域1名称]：[具体内容描述]，用[展现形式]组织，配手绘[图标]
2. [区域2名称]：[具体内容描述]，用[展现形式]组织，配手绘[图标]
...

[风格约束]：纯白背景。手绘风格，蓝色墨水笔线条。标题蓝色，内容用蓝色手绘线条框组织，不要实色填充背景。用手绘小图标代替emoji。颜色克制，主要使用蓝色，最多1个[点缀色]点缀。不要画页码、系列名、网格线、徽章。
```

#### 标题规范

| 规则 | 说明 | 示例 |
|------|------|------|
| 必须加"实战案例："前缀 | 所有实战类页面 | "实战案例：用豆包做一个歌词日历网页！" |
| 必须指定超大号 | 标题字号明显大于正文 | "蓝色超大号粗体（48px）" |
| 必须对比正文字号 | 明确说明与正文差异 | "字号明显大于正文（18px）" |
| 标题位置 | 位于页面顶部 | "位于页面顶部" |

#### 内容撑满技巧

当内容较少时，prompt 中必须包含以下描述之一：
- "页面内容从上到下均匀分布，不要集中在上半部分"
- "底部添加补充说明区域，用蓝色手绘线条框组织"
- "主视觉元素适当放大，占据页面主要垂直空间"
- "如果内容较少，在底部添加装饰性总结框或提示框，撑满整图"

### Step 3：调用 ImageGen

1. 生图前清理 vibe_images/：`cd /Users/kathyzhang/Documents/vibe_images && rm -f P*.png 2>/dev/null`
2. 调用 ImageGen 生成图片
3. 图片保存到 vibe_images/ 目录

### Step 4：Pipeline 处理

运行 template_pipeline.py 合成网格背景：

```bash
cd /Users/kathyzhang/Documents/kz-xhs-AIagent && python3 xhs-AIcontents/scripts/template_pipeline.py \
  --input-dir /Users/kathyzhang/Documents/vibe_images/ \
  --output-dir "/Users/kathyzhang/Documents/kz-xhs-AIagent/xhs-AIcontents/xhs-post/{post-path}/temporary/P{N}-{主题}" \
  --total-pages {总页数} \
  --series-name "{系列名}" \
  --pages {当前页码} \
  --no-badges
```

**说明：**
- `--no-badges` 为默认参数（徽章已废弃）
- Pipeline 自动扫描 output_dir 中已有最高版本号并 +1
- 处理后图片存入 `temporary/P{N}-{主题}/`

### Step 5：生成后确认流程（硬性规则）

1. **报告存放位置：** 明确告知用户图片已存入哪个页面文件夹及版本号
2. **请求确认：** 询问用户"是否将此图片复制到 final version/？"
3. **等待确认：** 必须等待用户确认后才执行复制

### Step 6：反馈闭环（硬性规则）

当用户给出修改意见或否决方案时：
1. 将用户意见追加到 `memory/feedback-log.md`
2. 如果用户确认了某个设计方向，追加到 `memory/decisions-log.md`
3. 同类反馈出现 2 次及以上时，提取为规则写入 workflow 或 agent 配置

## Prompt 失败经验

| 错误 | 表现 | 正确做法 |
|------|------|---------|
| 结构标签当内容 | ImageGen 把"区域1"等标签画出来 | 用自然语言描述位置："左上方"、"底部"等 |
| 底部留白过多 | 内容集中在上半部分 | 明确写"均匀分布"，添加底部装饰区域 |
| 标题不够醒目 | 标题颜色淡或模糊 | 写"醒目的深蓝色粗体大号标题" |
| 临时文件混乱 | vibe_images/ 积累旧文件 | 生图前清理 vibe_images/，保留 temporary/ 历史版本 |

|------|------------|
| 日期/日历 | "手绘日历图标" |

| 完成/推荐 | "手绘对勾图标（绿色点缀）" |
| 步骤引导 | "手绘箭头图标" |

## 文件管理规范

1. **vibe_images/：** 仅存放 ImageGen 原始输出，生图前清理旧文件
2. **temporary/：** 按页面分文件夹 `P{N}-{主题}/`，保留所有历史版本
3. **final version/：** 仅存用户确认的最终成品，文件名不带版本号
4. **严禁删除 temporary/ 中的历史版本**

## 已验证成功的 Prompt 示例

```
实战案例：用豆包做一个歌词日历网页！蓝色超大号粗体（48px）标题，字号明显大于正文（18px），位于页面顶部。

页面内容分为2个区域：
1. 浏览器预览窗口：模拟浏览器窗口，地址栏显示 file:///calendar.html，窗口内展示歌词日历效果（每日经典歌词日历标题、2026年4月3日日期、歌词卡片显示歌名和歌手），用蓝色手绘线条框组织，配手绘浏览器图标
2. 功能清单：用绿色手绘对勾图标列出5个功能点（显示当前日期：2026-04-03、展示当日歌词：歌名+歌手+歌词、左右翻页：切换前一天/后一天、风格切换：摇滚/港台/周杰伦/民谣、切换风格后，歌词数据全部更新）

纯白背景。手绘风格，蓝色墨水笔线条。标题蓝色，内容用蓝色手绘线条框组织，不要实色填充背景。用手绘小图标代替emoji。颜色克制，主要使用蓝色，最多1个绿色点缀。不要画页码、系列名、网格线、徽章。
```
