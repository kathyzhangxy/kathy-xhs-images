# ImageGen Prompt

> 📝 P3-V5 使用的完整 ImageGen prompt

## 完整 Prompt

```
纯白背景。极简手绘线条风（notion风格），蓝色墨水笔线条，手绘感强。

页面内容分为左右两个手机屏幕，展示"输入→输出"流程：

左侧手机屏幕：模拟豆包 App 界面
- 顶部状态栏：显示时间"14:32"、信号图标、电池图标（手绘风格）
- 标题栏：蓝色手绘字体"豆包"
- 中间内容区：蓝色渐变对话气泡，展示用户发送的结构化 Prompt
  * 角色设定："10年前端工程师"
  * 功能需求：7条（标题、日期、歌词卡片、翻页按钮、风格选择器、切换更新、翻页查看）
  * 样式要求：6条（渐变背景、卡片样式、宽度、按钮、选择器、移动端适配）
  * 数据要求：30首歌曲不重复
  * 输出格式：完整HTML代码
  用深蓝色代码块样式展示，浅色等宽字体，紧凑排列
- 底部操作栏：蓝色手绘发送按钮图标

右侧手机屏幕：模拟豆包 App 界面
- 顶部状态栏：显示时间"14:32"、信号图标、电池图标（手绘风格）
- 标题栏：蓝色手绘字体"豆包"
- 中间内容区：白色圆角窗口，展示豆包返回的代码
  * 顶部显示"预览 | 代码"标签（手绘风格）
  * 窗口内展示HTML代码片段（<button>、<main class="lyrics-grid">、<script>等）
  * 代码用等宽字体，蓝色高亮关键字
  * 底部显示"正在生成代码..."加载提示
- 底部操作栏：蓝色手绘输入框图标

中间用粗手绘曲线箭头从左手机指向右手机，橙色，带手绘曲线感。

底部区域：手绘风格蓝色虚线框，框内手绘灯泡图标 + "发送后等待10-30秒，豆包会生成完整的HTML代码"

点缀元素：
- 右上角：手绘橙色小星星 ⭐
- 左侧手机屏幕左边：手绘紫色对话气泡图标 💬
- 右侧手机屏幕右边：手绘橙色代码标签图标 </>
- "10-30秒"文字用橙色手绘圆圈圈起来

页面内容从上到下均匀分布，不要集中在上半部分。
```

## 关键成功要素

1. **手机屏幕模拟**：必须明确描述状态栏、标题栏、内容区、底部操作栏四个层次
2. **手绘风格一致性**：所有元素都要有手绘感（虚线框、曲线箭头、手绘图标）
3. **颜色分布**：主色调蓝色（手机界面）+ 橙色点缀（箭头、星星、代码标签、时间圈）+ 紫色点缀（对话气泡）
4. **点缀数量**：4-5个点缀元素，分布在页面四周，不要集中在一个位置
5. **内容丰满度**：Prompt 内容要详细列出，不能过于简略，确保画面信息密度
