# ImageGen Prompt 方法论

> 用于手绘田字格模板（handdrawn-grid-blue）的 prompt 撰写规范
> 更新日期：2026-04-04
> 版本：v1.0

---

## 一、核心原则

### 1.1 确定性 vs 创意性的边界

| 任务类型 | 处理方式 | 示例 |
|---------|---------|------|
| **确定性任务** | 用代码（Pillow/Pipeline） | 网格背景、徽章叠加、尺寸调整 |
| **创意性任务** | 用 AI（ImageGen） | 页面布局、内容排版、手绘风格 |

**关键认知**：ImageGen 的本质是"用文字描述一幅画，让 AI 画出来"。prompt 的质量直接决定画面质量。

### 1.2 Prompt 的三层结构

每个 prompt 都包含三个层次：

```
第一层：内容层 —— 画什么（标题、模块、文字）
第二层：风格层 —— 怎么画（手绘风、蓝色墨水、纯白背景）
第三层：约束层 —— 不画什么（不要emoji、不要实色、不要网格）
```

**错误做法**：只有内容层，缺少风格和约束层 → 风格跑偏
**正确做法**：三层完整 → 风格可控

## 二、页面类型判断

生图前第一步：判断当前页面是 Cover 还是 Inner。

| 类型 | 判断依据 | 参考案例 | 核心目标 | 信息密度 |
|------|---------|---------|---------|---------|
| **Cover** | C2 中 P0 大字报封面 | `good/cover/` | 3秒吸引：超大标题+关键词高亮 | 低（极简） |
| **Inner** | C2 中 P1-P11 内容页 | `good/inner/` | 信息传达：模块化+图标辅助 | 高（丰满） |

## 三、Cover Prompt 撰写方法

### 3.1 结构公式

```
[金句文字]。手绘风大字报，[颜色]纯色背景，
超大字号居中，[关键词]用[颜色]色块高亮。
极简，无多余元素。手绘风格，蓝色墨水笔线条。
不要画网格、页码、徽章、emoji。
```

### 3.2 撰写要点

| 要点 | 说明 | 反例 |
|------|------|------|
| **金句优先** | 必须是有冲击力的话 | "这是一篇关于AI的教程" ❌ |
| **背景色** | 柔和纯色（浅蓝/浅粉/浅黄） | 不要网格、渐变 |
| **关键词** | 1-2个，指定颜色 | 不要超过3个高亮 |
| **极简** | 不要描述装饰元素 | 不要画星星/花/线条 |

### 3.3 示例

```
不会代码？AI帮你5分钟搭网页！手绘风大字报，浅蓝色纯色背景，
超大字号居中显示标题，"AI"用黄色色块高亮，"5分钟"用粉色色块高亮。
极简设计，无多余元素。手绘风格，蓝色墨水笔线条。
不要画网格、页码、徽章、emoji。
```

## 四、Inner Prompt 撰写方法

### 4.1 结构公式

```
[标题]，蓝色[字号]粗体，位于页面[位置]。

内容分为[N]个区域：
1. [区域名称]：[具体文字内容]，用[展现形式]组织，配手绘[图标]图标
2. [区域名称]：[具体文字内容]，用[展现形式]组织，配手绘[图标]图标
3. ...

纯白背景。手绘风格，蓝色墨水笔线条。
标题蓝色，内容用蓝色手绘线条框，不要实色填充。
用手绘小图标代替emoji。颜色克制，主要蓝色，最多1个[颜色]点缀。
不要画页码、系列名、网格线、徽章。
```

### 4.2 撰写要点详解

#### 4.2.1 标题

| 要点 | 说明 |
|------|------|
| **必须指定颜色** | "蓝色大号粗体"，不能只说"大标题" |
| **必须指定位置** | "位于页面顶部" 或 "居中显示" |
| **建议指定字号** | "大号/中号/小号" 或 "36px/24px/18px" |

#### 4.2.2 内容模块化

**核心原则**：把页面切成 2-4 个区域，每个区域独立描述。

| 要素 | 说明 |
|------|------|
| **数量** | 2-4个区域，太少则空，太多则乱 |
| **每个区域** | 名称 + 具体内容 + 展现形式 + 图标 |
| **区域关系** | 明确是纵向排列、左右分栏、还是嵌套 |

#### 4.2.3 展现形式选择

根据内容性质选择最合适的展现形式：

| 展现形式 | 适用场景 | 描述关键词 | 示例 |
|---------|---------|-----------|------|
| **卡片列表** | 多个并列概念/功能 | 蓝色手绘线条框、纵向排列、圆角矩形 | 功能清单 |
| **时间线** | 步骤/流程/阶段 | 纵向虚线连接、编号圆圈、步骤图标 | 操作步骤 |
| **对比框** | 修改前后/A vs B | 左右分栏、箭头连接、标签 | 调试前后 |
| **流程图** | 数据流向/逻辑关系 | 手绘箭头、节点框、方向标注 | 数据处理 |
| **对话气泡** | 用户输入/AI回复 | 白色对话气泡、左/右对齐 | Prompt对话 |
| **代码块** | Prompt/代码片段 | 深蓝底色矩形、等宽文字 | 代码展示 |
| **浏览器预览** | 网页效果示意 | 模拟浏览器窗口、地址栏 | 网页效果 |
| **勾选清单** | 功能清单/检查项 | 手绘勾选框、短文字 | 功能说明 |
| **要点列表** | 总结/知识点 | 手绘圆点+短文字 | 总结页 |

#### 4.2.4 图标使用

**核心原则**：每个关键信息点配一个手绘小图标，辅助理解。

| 图标 | prompt 描述 | 适用场景 |
|------|------------|---------|
| 🎯 靶心 | 手绘靶心图标 | 目标/重点 |
| 💡 灯泡 | 手绘灯泡图标 | 技巧/想法 |
| ➡️ 箭头 | 手绘箭头图标 | 流程/指向 |
| 💬 对话框 | 手绘对话气泡 | 对话/输入 |
| ☑️ 勾选框 | 手绘勾选框 | 清单/完成 |
| 📅 日历 | 手绘日历图标 | 日期/时间 |
| 🎵 音符 | 手绘音符图标 | 音乐/歌词 |
| ⚙️ 齿轮 | 手绘齿轮图标 | 设置/工具 |
| ✅ 对勾 | 手绘对勾图标 | 完成/推荐 |
| ⚠️ 警告 | 手绘警告三角 | 注意/问题 |
| 📖 书本 | 手绘书本图标 | 知识/学习 |
| </> 代码 | 手绘代码括号 | 编程/技术 |

#### 4.2.5 信息密度

**这是最重要的要点之一**。页面不能空，也不能挤。

| 密度问题 | 原因 | 解决 |
|---------|------|------|
| **页面太空** | 内容描述太笼统，没具体到元素级别 | 每个区域描述到"有什么文字+什么图标+什么框" |
| **页面太挤** | 区域太多或文字太长 | 控制在2-4个区域，文字精简 |

**判断标准**：参考 good/inner/ 中的案例，页面应该有"恰到好处的丰富感"。

### 4.3 完整示例

```
今天用豆包做一个歌词日历网页！蓝色大号粗体标题，位于页面顶部。

页面内容分为3个区域：
1. 浏览器预览窗口：模拟Chrome浏览器窗口，蓝色手绘细线边框，
   窗口顶部有地址栏显示"file:///calendar.html"，窗口内展示歌词日历效果：
   顶部显示"2026年4月3日 星期五"，中间是歌词卡片"《晴天》周杰伦 - 故事的小黄花"，
   底部有左右翻页箭头按钮
2. 功能清单：4个手绘勾选框纵向排列在浏览器窗口右侧，每个框配手绘小图标：
   手绘日历图标+"显示当前日期"，手绘音符图标+"展示歌词"，
   手绘双向箭头+"左右翻页"，手绘标签图标+"风格切换"
3. 底部提示：手绘对勾图标（绿色点缀）+"零代码，3步搞定"，蓝色中号文字

纯白背景。手绘风格，蓝色墨水笔线条。
标题蓝色，内容用蓝色手绘线条框组织，不要实色填充背景。
用手绘小图标代替emoji。颜色克制，主要使用蓝色，最多1个绿色点缀。
不要画页码、系列名、网格线、徽章。
```

## 五、Prompt 调试技巧

### 5.1 迭代策略

| 迭代版本 | 目标 | 调整方向 |
|---------|------|---------|
| **V1** | 验证整体框架 | 布局是否正确、模块是否清晰 |
| **V2** | 调整内容密度 | 增加/减少元素、调整文字量 |
| **V3** | 微调细节 | 图标位置、颜色比例、间距感 |

### 5.2 常见问题与修复

| 问题 | 原因 | Prompt 修复方法 |
|------|------|----------------|
| 标题是黑色 | 没指定颜色 | 加"蓝色大号粗体" |
| 页面太空 | 描述太笼统 | 增加具体元素描述 |
| 有实色背景 | 没禁止填充 | 加"不要实色填充" |
| 有 emoji | 没禁止 | 加"用手绘小图标代替emoji" |
| 颜色太多 | 没限制 | 加"最多1个XX色点缀" |
| 有网格线 | 描述了网格 | 删除网格描述 |
| 文字不可读 | ImageGen 文字能力限制 | 减少文字量，用图标替代 |
| 布局混乱 | 区域描述不清晰 | 明确区域位置和关系 |

### 5.3 Prompt 调试原则

1. **每次只改一个问题**：不要同时改颜色+布局+密度
2. **改了什么要记录**：V1→V2 改了哪些描述
3. **好的 V1 来自好的思路**：生图思路对了，V1 就能达到 70 分

## 六、经验沉淀：从 Bad Cases 学到的

### 6.1 最惨痛的教训（P6-OpenClaw 迭代 8 次）

| 版本 | 问题 | 根因 | 规则 |
|------|------|------|------|
| V1-V2 | 彩色实色卡片+emoji | 没参考 good case | 生图前必须看参考图 |
| V3 | 双重网格 | prompt 描述了背景网格 | 不要描述网格 |
| V4 | "4个Agent"写错 | 没核对内容 | 数字必须准确 |
| V5-V6 | 顶部大片空白 | 用了英文留白描述 | 禁止英文留白描述 |
| V7-V8 | 标题是黑色 | 没指定颜色 | 标题必须指定蓝色 |

### 6.2 提炼为硬性规则

| 规则 | 违反后果 | 检查时机 |
|------|---------|---------|
| 标题必须指定蓝色 | 标题变黑色 | Prompt 撰写时 |
| 不使用 emoji | 画风不统一 | Prompt 撰写时 |
| 不使用实色填充 | 风格跑偏 | Prompt 撰写时 |
| 颜色 ≤ 1主色+1点缀 | 花花绿绿 | Prompt 撰写时 |
| 内容密度要够 | 页面太空 | 生图思路确认时 |
| 纯白背景，不描述网格 | 双重网格 | Prompt 撰写时 |
| 生图前看 good case | 风格跑偏 | Step 2 时 |

## 七、生图完整流程

```
Step 1: 判断页面类型
  → Cover → 参考 good/cover/
  → Inner → 参考 good/inner/

Step 2: 输出「生图思路」→ 主管确认
  → 布局框架 + 内容模块 + 视觉元素 + 参考案例

Step 3: 基于思路撰写 Prompt
  → 套用 Cover/Inner 模板
  → 逐项过检查清单

Step 4: ImageGen 生成
  → size: 1024x1536（竖版）

Step 5: 文件处理
  → 移动到 temporary/P{N}-{页面名}/P{N}-{页面名}-V{版本}.png
  → 删除 vibe_images/ 源文件
  → 运行 pipeline（--no-badges）

Step 6: 主管确认
  → 满意 → final version/
  → 不满意 → 记录问题 → 调整 prompt → V{N+1}
```

## 八、检查清单

每次调用 ImageGen 前，逐项检查：

### 生图前
- [ ] 已判断页面类型（Cover / Inner）
- [ ] 已输出「生图思路」并获得主管确认
- [ ] 已参考对应的 good case

### Prompt 撰写
- [ ] 标题颜色明确指定为蓝色（"蓝色大号粗体"）
- [ ] 不使用 emoji（"用手绘小图标代替"）
- [ ] 不使用实色填充（"不要实色填充背景"）
- [ ] 颜色 ≤ 1 主色（蓝）+ 1 点缀色
- [ ] 内容描述具体到元素级别
- [ ] 纯白背景（不描述网格）
- [ ] 无需顶部留白

### 生图后
- [ ] 标题是蓝色
- [ ] 没有 emoji
- [ ] 没有实色填充
- [ ] 页面内容丰满
- [ ] 文件已正确移动和命名
- [ ] Pipeline 已成功执行

## 十、Prompt 模板库

每次生图前，先查 `knowledge/prompt-templates/inner-prompts/` 中对应类型的模板文件。

| 模板文件 | 适用页面 | 链接 |
|---------|---------|------|
| 01-任务介绍.md | P1/P3/P10 等介绍任务的页面 | `inner-prompts/01-任务介绍.md` |
| 02-技巧说明.md | P2 等技巧/方法教学页面 | （待创建） |
| 03-浏览器预览.md | 带网页截图的页面 | （待创建） |
| 04-操作步骤.md | P5 等分步教程页面 | （待创建） |
| 05-问题调试.md | P6-P9 等调试/问题页面 | （待创建） |
| 06-总结预告.md | P11 等总结+预告页面 | （待创建） |

**使用方式**：
1. 判断当前页面类型
2. 找到对应模板，复制框架
3. 替换具体内容（标题、区域描述、图标）
4. 生成
