# 页面类型：任务介绍 / 实战案例

> 适用页面：Post 中介绍本次任务的页面（通常为 P1 或 P3）
> 更新日期：2026-04-04

---

## 通用 Prompt 框架

```
实战案例：[用XX做XX]！蓝色超大号粗体（48px）标题，字号明显大于正文（18px），位于页面顶部。

页面内容分为[N]个区域：
1. [浏览器预览窗口/效果展示]：[具体描述]，用[展现形式]组织，配手绘[图标]
2. [功能清单/文字说明]：[具体描述]，用[展现形式]组织，配手绘[图标]
3. [底部提示/操作引导]：[具体描述]

纯白背景。手绘风格，蓝色墨水笔线条。标题蓝色，内容用蓝色手绘线条框组织，不要实色填充背景。用手绘小图标代替emoji。颜色克制，主要使用蓝色，最多1个[颜色]点缀。不要画页码、系列名、网格线、徽章。
```

## 标题规范

| 规则 | 说明 | 示例 |
|------|------|------|
| **必须加"实战案例："前缀** | 所有实战类页面的标题格式 | "实战案例：用豆包做一个歌词日历网页！" |
| **必须指定超大号** | 标题字号明显大于正文 | "蓝色超大号粗体（48px）" |
| **必须对比正文字号** | 明确说明与正文的字号差异 | "字号明显大于正文（18px）" |
| **标题位置** | 位于页面顶部 | "位于页面顶部" |

## 已验证成功的 Prompt（P1-V8 通过）

```
实战案例：用豆包做一个歌词日历网页！蓝色超大号粗体（48px）标题，字号明显大于正文（18px），位于页面顶部。

页面内容分为2个区域：
1. 浏览器预览窗口：模拟浏览器窗口，地址栏显示 file:///calendar.html，窗口内展示歌词日历效果（每日经典歌词日历标题、2026年4月3日日期、歌词卡片显示歌名和歌手），用蓝色手绘线条框组织，配手绘浏览器图标
2. 功能清单：用绿色手绘对勾图标列出5个功能点（显示当前日期：2026-04-03、展示当日歌词：歌名+歌手+歌词、左右翻页：切换前一天/后一天、风格切换：摇滚/港台/周杰伦/民谣、切换风格后，歌词数据全部更新）

纯白背景。手绘风格，蓝色墨水笔线条。标题蓝色，内容用蓝色手绘线条框组织，不要实色填充背景。用手绘小图标代替emoji。颜色克制，主要使用蓝色，最多1个绿色点缀。不要画页码、系列名、网格线、徽章。
```

**关键成功要素：**
- ✅ 严格遵循模板的三段式结构（标题 → 区域描述 → 风格约束）
- ✅ "纯白背景"（网格由 pipeline 自动添加，不在 prompt 中描述）
- ✅ 区域描述简洁，不指定过多细节
- ✅ 风格约束完整（纯白背景 + 手绘风格 + 蓝色墨水笔 + 不要实色填充 + 不要页码系列名网格线徽章）

## Good Case 色彩和点缀经验总结

### 基于 P3-Token
- 进度条/数据条：浅橙色渐变，配小图标
- 装饰小图标：手绘小星星、代码标签、对话气泡等（橙色点缀）
- 关键词高亮：重要数字或关键词用橙色/桃色方块或下划线
- 手绘小图标：每个要点前面配小圆点/图标

### 基于 P4-ClaudeCode-V2（最佳参考）
**氛围点缀方式：**
- **渐变背景卡片**：输入/输出区域使用浅蓝渐变背景卡片（不是纯色，有微妙渐变）
- **屏幕内嵌内容**：左侧黑色终端窗口（代码风格），右侧浏览器窗口（产品效果），形成"输入→输出"的视觉叙事
- **手绘箭头连接**：粗手绘箭头从输入指向输出，带手绘曲线感
- **对比框设计**：底部 VS 对比框用双线框+手绘角标，区分两个对比对象
- **小图标点缀**：右上角小星星图标（橙色）、左侧小图标（紫色/蓝色）
- **色彩层次**：主色蓝色 + 点缀橙色 + 次要色紫色，层次分明但不花哨

### 内容撑满整图的技巧（避免下方留白过多）
**问题：** 内容较少时，ImageGen 生成的内容集中在上半部分，下方大面积留白，页面显得空洞。

**解决方案：**
1. **增加装饰性底部区域**：如 P4 的 VS 对比框、总结卡片、提示框等
2. **垂直分布内容**：不要把所有内容挤在上半部分，标题、主内容、补充说明/底部区域均匀分布
3. **添加补充信息区**：如"关键提示"、"为什么这样做"、"注意事项"等文字块
4. **底部装饰条**：手绘线条分隔的底部区域，放总结性文字或引导语
5. **放大主视觉元素**：如果只有一个主视觉（如手机/浏览器窗口），让它更大，占据更多垂直空间

**Prompt 中的描述方式：**
- "页面内容从上到下均匀分布，不要集中在上半部分"
- "底部添加补充说明区域，用蓝色手绘线条框组织"
- "主视觉元素（手机/浏览器窗口）适当放大，占据页面主要垂直空间"
- "如果内容较少，在底部添加装饰性总结框或提示框，撑满整图"

**约束：**
- 点缀色最多 2-3 个元素，不能多
- 点缀色以橙色/桃色为主，与蓝色主色调形成对比
- 可以添加浅渐变背景卡片增加层次感
- 不要实色填充大区域，只在局部点缀

## Prompt 失败经验总结

### 错误 1：把结构标签当成内容画出来
**表现：** V5、V6 中，ImageGen 把"区域1（上部）- 输入："、"区域2（中部）- 转换与输出："等结构描述当成了实际页面内容画出来。
**原因：** Prompt 中用了"区域1"、"区域2"这样的标签式描述，ImageGen 理解为页面文字内容。
**正确做法：** 用自然语言描述位置关系，如"左上方"、"右上方"、"中间"、"底部"，不要用"区域X"、"模块X"等标签。参考 P4-ClaudeCode-V2 的简洁描述方式。

### 错误 2：底部留白过多
**表现：** V1-V4 中，内容集中在页面上半部分，下方大面积空白。
**原因：** 内容较少时，没有明确指示"撑满整图"和"均匀分布"。
**正确做法：** 
- Prompt 中明确写"页面内容从上到下均匀分布，不要集中在上半部分"
- 添加底部装饰区域（关键提示框、总结框、VS 对比框等）
- 放大主视觉元素占据更多垂直空间

### 错误 3：标题颜色不够醒目
**表现：** V1-V3 中标题颜色淡或模糊。
**原因：** Prompt 中只写了"蓝色"，没有强调"醒目的深蓝色粗体"。
**正确做法：** 写"醒目的深蓝色粗体大号标题"，明确强调颜色和粗细。

### 错误 4：右侧缺少上下文延续
**表现：** V1-V2 中，右侧手机屏幕直接显示结构化 Prompt，用户看不出是从左边得出来的。
**原因：** 没有描述"用户输入"的延续显示。
**正确做法：** 在右侧屏幕顶部显示"用户输入：..."小字，加向下箭头引导到结构化内容。

### 错误 5：临时文件管理混乱
**表现：** vibe_images/ 和 temporary/ 中积累了大量中间文件。
**原因：** 
- pipeline 扫描整个 input-dir，vibe_images/ 中的旧文件会被重复处理
- 生成新版本时错误地删除了 temporary/ 中的历史版本
**正确做法：**
- 生图前清理 vibe_images/：`cd vibe_images && rm -f P*.png`
- 严禁删除 temporary/ 中的历史版本，所有 V 版本都要保留

## 踩过的坑

| 问题 | 错误做法 | 正确做法 |
|------|---------|---------|
| 标题缺少"实战案例"前缀 | "今天用豆包做一个歌词日历网页！" | "实战案例：用豆包做一个歌词日历网页！" |
| 标题和正文字号差不多 | "蓝色大号粗体标题" | "蓝色超大号粗体（48px），字号明显大于正文（18px）" |
| 浏览器地址栏写复杂URL | "file:///Users/xxx/calendar.html" | "file:///calendar.html"（简化） |
| 歌词卡片文字太多 | "《晴天》周杰伦的故事的小黄花" | "故事的小黄花"（精简到一句歌词） |
| 底部提示文字冗余 | "零代码文字" | "零代码"（去掉多余的"文字"二字） |
