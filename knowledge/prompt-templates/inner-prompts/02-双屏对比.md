# 页面类型：双屏对比（输入→输出）

> 适用页面：展示"发送指令→AI生成"过程的页面
> 参考案例：P4-ClaudeCode-V2（最佳参考）
> 更新日期：2026-04-05

---

## 标题模块 Prompt 写法

```
[主标题] — [副标题关键词]！蓝色超大号粗体（48px）标题，字号明显大于正文（18px），位于页面顶部。
```

**要点：**
- 主标题 + 破折号 + 副标题关键词
- 必须指定"蓝色超大号粗体"
- 必须说明"字号明显大于正文（18px）"
- 必须说明"位于页面顶部"

---

## 左右双屏模块 Prompt 写法

```
页面内容分为左右两个区域：
1. [左侧标题]：[深色/白色]圆角矩形窗口，顶部标题栏显示"[标题文字]"，窗口内[具体内容描述]，用[展现形式]组织，配手绘[图标]图标
2. [右侧标题]：[深色/白色]圆角矩形窗口，顶部标题栏显示"[标题文字]"，窗口内[具体内容描述]，用[展现形式]组织，配手绘[图标]图标

中间用粗手绘曲线箭头从左指向右连接，带手绘曲线感，箭头旁标注"[连接文字]"。
```

**要点：**
- 用"圆角矩形窗口"描述设备屏幕，不要用"对话气泡"
- 左侧可用深色（模拟终端/深色模式），右侧用白色
- 每个窗口必须有"顶部标题栏"
- 窗口内内容用列表/卡片/代码等展现形式组织
- 中间箭头用"粗手绘曲线箭头"，强调"手绘曲线感"

---

## 底部模块 Prompt 写法

```
底部区域：[双线手绘框]，分为[N]个区域，中间用"[分隔文字]"大字号分隔。
左侧：[内容描述]
中间：[内容描述]
右侧：[内容描述]
```

**要点：**
- 用"双线手绘框"描述 VS 对比框
- 明确分隔文字（如"VS"、"→"）
- 每个区域独立描述

---

## 点缀描述方式

```
手绘风格，蓝色墨水笔线条。
颜色层次：主色蓝色 + 点缀橙色 + 次要色紫色。
点缀元素：
- 右上角手绘小星星图标（橙色点缀）
- 左侧[位置]小图标（紫色/蓝色点缀）
- 关键词/数字用橙色/桃色方块高亮
- 进度条/数据条用浅橙色渐变，配小图标

不要实色填充大区域，只在局部点缀。
点缀色最多 2-3 个元素，不能多。
```

**要点：**
- 明确色彩层次：主色 + 点缀 + 次要色
- 具体描述每个点缀元素的位置和颜色
- 强调"不要实色填充大区域"
- 限制点缀数量：最多 2-3 个

## 完整 Prompt 模板

```
[主标题] — [副标题]！蓝色超大号粗体（48px）标题，字号明显大于正文（18px），位于页面顶部。

页面内容分为左右两个区域：
1. [左侧标题]：[深色]圆角矩形窗口，顶部标题栏显示"[标题]"，窗口内[内容]，用[形式]组织，配手绘[图标]图标
2. [右侧标题]：[白色]圆角矩形窗口，顶部标题栏显示"[标题]"，窗口内[内容]，用[形式]组织，配手绘[图标]图标

中间用粗手绘曲线箭头从左指向右连接，带手绘曲线感，箭头旁标注"[文字]"。

底部区域：[描述]。

纯白背景。手绘风格，蓝色墨水笔线条。
颜色层次：主色蓝色 + 点缀橙色 + 次要色紫色。
右上角手绘小星星图标（橙色点缀），[其他点缀描述]。
不要实色填充大区域，只在局部点缀。点缀色最多2-3个元素。
用手绘小图标代替emoji。不要画页码、系列名、网格线、徽章。
页面内容从上到下均匀分布，不要集中在上半部分。
```
